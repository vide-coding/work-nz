# 任务看板模块 - 第 2 步：子任务

## 目标

在第 1 步基础上，添加子任务功能。子任务只能在自己的父任务下操作，不支持跨列拖动。

## 数据库变更

tasks 表已支持（Step 1 已建），新增字段在 Step 1 已加入：
- `parent_id TEXT` - 父任务 ID
- `is_completed INTEGER` - 任务完成状态

新增 API：

| 命令 | 签名 | 说明 |
|------|------|------|
| `task_list_children` | `(parent_id: String) -> Vec<Task>` | 获取某任务的子任务列表 |
| `task_toggle_complete` | `(id: String) -> Task` | 切换子任务的 is_completed 状态 |

## 前端变更

### Task 类型扩展

```typescript
interface Task {
  // ... Step 1 字段
  parentId?: string           // 有值则为子任务
  isCompleted: boolean        // 子任务完成状态
  childTasks?: Task[]         // 前端展开时加载子任务
}
```

### 组件变更

#### TaskCard.vue 变更

子任务卡片展示内容：
- 勾选框（左侧，点击切换 is_completed）
- 标题（已完成的显示删除线）
- 不显示优先级、指派人（Step 1 已确定子任务只需标题+完成状态）

父任务卡片新增：
- 子任务进度指示器（如 "2/5" 显示完成数）
- 展开/折叠按钮（右侧，点击展开子任务列表）

```
┌──────────────────────────────────┐
│ ☐ 子任务 1                  [−]  │
├──────────────────────────────────┤
│ ☑ 子任务 2（已完成，删除线）      │
├──────────────────────────────────┤
│ ☐ 子任务 3                      [+ 添加子任务] │
└──────────────────────────────────┘
```

### 详情面板变更

在详情面板底部添加子任务区域：
- 子任务列表（可勾选完成）
- "添加子任务" 按钮（输入框 + 回车创建）
- 删除子任务（hover 显示删除按钮）

### 子任务业务逻辑

1. **创建子任务**：默认 `is_completed=false`，无优先级/指派字段
2. **切换完成状态**：调用 `task_toggle_complete` API
3. **删除父任务**：级联删除所有子任务（Rust ON DELETE CASCADE 或手动处理）
4. **展开/折叠状态**：前端状态管理，不需要持久化

### 拖拽限制

- 子任务只能在其父任务的子任务列表内上下拖动排序
- 子任务不能跨列拖动（没有 status 字段变化入口）

## 交付标准

- [ ] 父任务卡片显示子任务数量和完成进度
- [ ] 点击展开可查看和操作子任务列表
- [ ] 子任务可新增（输入标题+回车）
- [ ] 子任务可勾选完成/未完成
- [ ] 子任务可删除
- [ ] 子任务可在列表内拖动排序
- [ ] 删除父任务时自动删除所有子任务
