## Why

用户需要一个直观的文件资源管理功能来管理项目内的文件。当前的项目页面缺乏文件操作能力，用户需要能够创建、组织和操作项目内的文件，同时支持拖拽等现代交互方式提升效率。

## What Changes

- 新增文件列表组件，支持展示项目目录内的文件和文件夹
- 支持创建文件和文件夹
- 支持通过系统默认程序打开文件
- 支持拖拽外部文件到应用目录内
- 支持重命名文件和文件夹
- 支持删除文件和文件夹（带确认提示）

## Capabilities

### New Capabilities

- `file-manager`: 项目页面文件管理模块
  - 文件/文件夹列表展示
  - 创建文件和文件夹
  - 打开文件（调用系统默认程序）
  - 拖拽文件到项目目录
  - 重命名文件/文件夹
  - 删除文件/文件夹

## Impact

- 新增前端组件：`src/views/` 项目页面
- 新增 Tauri 命令：文件系统操作（创建、读取、删除、重命名）
- 新增 Pinia store：文件状态管理
- UI 改动：项目页面新增文件管理面板
