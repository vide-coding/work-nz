## 新增需求

### 需求：任务创建和管理

Task 模块应提供在目录内创建、更新和删除任务的能力。

#### 场景：任务创建

- **当** 用户创建带有标题和描述的新任务
- **那么** 系统应以唯一标识符存储任务
- **并且** 任务应具有创建时间戳
- **并且** 任务应与当前目录关联

#### 场景：任务更新

- **当** 用户修改现有任务
- **那么** 系统应更新任务字段
- **并且** 系统应记录最后修改时间戳

#### 场景：任务删除

- **当** 用户删除任务
- **那么** 系统应从目录中移除任务
- **并且** 系统应提供带归档的软删除选项

### 需求：任务状态跟踪

Task 模块应支持任务的状态跟踪，包括可自定义的状态工作流。

#### 场景：状态更新

- **当** 用户更改任务状态
- **那么** 系统应更新任务状态
- **并且** 系统应记录状态更改时间戳

#### 场景：自定义状态配置

- **当** 用户配置 Task 模块
- **那么** 用户应能够定义自定义状态值
- **并且** 用户应能够定义状态转换（哪些状态可以转换到哪些状态）
- **并且** 系统应提供默认状态集（To Do、In Progress、Done）

### 需求：任务优先级级别

Task 模块应支持任务的优先级级别。

#### 场景：优先级分配

- **当** 用户创建或更新任务
- **那么** 用户应能够设置优先级级别
- **并且** 系统应提供默认优先级级别（Low、Medium、High、Urgent）
- **并且** 用户应能够配置自定义优先级级别

### 需求：任务分配

Task 模块应支持将任务分配给用户。

#### 场景：任务分配

- **当** 用户创建或更新任务
- **那么** 用户应能够将任务分配给用户
- **并且** 分配应包含被分配者标识符和分配时间戳

### 需求：任务列表和过滤

Task 模块应提供带过滤和排序能力的任务列表视图。

#### 场景：任务列表视图

- **当** 用户查看启用了 Task 模块的目录
- **那么** 系统应显示该目录中的任务列表
- **并且** 列表应显示任务标题、状态、优先级、负责人和截止日期

#### 场景：任务过滤

- **当** 用户过滤任务列表
- **那么** 系统应支持按状态、优先级、负责人和日期范围过滤
- **并且** 系统应支持与 AND 逻辑组合的多个过滤条件

#### 场景：任务排序

- **当** 用户排序任务列表
- **那么** 系统应支持按优先级、截止日期、创建日期和状态排序
- **并且** 用户应能够选择升序或降序

### 需求：任务模板

Task 模块应支持用于快速创建带预设值任务的任务模板。

#### 场景：任务模板创建

- **当** 用户将任务保存为模板
- **那么** 系统应将任务字段（标题除外）存储为模板
- **并且** 模板应具有名称和描述

#### 场景：从模板创建任务

- **当** 用户从模板创建任务
- **那么** 系统应使用模板值预填充任务字段
- **并且** 用户应在保存前能够修改字段

### 需求：任务配置选项

Task 模块应提供用于自定义任务行为的配置选项。

#### 场景：Task 模块配置

- **当** 用户配置启用了 Task 模块的目录
- **那么** 用户应能够定义自定义状态值和转换
- **并且** 用户应能够定义自定义优先级级别
- **并且** 用户应能够设置新任务的默认值
- **并且** 用户应能够配置任务模板
