## 新增需求

### 需求：目录模板定义

目录模板应由一组目录及其关联的模块和配置组成。

#### 场景：从现有目录创建模板

- **当** 用户将一组目录保存为模板
- **那么** 系统应捕获每个目录的名称、模块和配置
- **并且** 模板应以名称、描述和 scope 存储

### 需求：模板存储范围

模板应支持多种存储范围：local（用户特定）、project（项目内共享）和 official（系统提供）。

#### 场景：模板范围可见性

- **当** 用户浏览可用模板
- **那么** local 模板应仅对创建用户可见
- **并且** project 模板应对所有项目成员可见
- **并且** official 模板应对所有用户可见

### 需求：项目创建期间的模板应用

系统应允许在项目创建时选择模板以自动创建目录。

#### 场景：带模板的项目创建

- **当** 用户创建新项目
- **并且** 用户选择目录模板
- **那么** 系统应创建模板中定义的所有目录
- **并且** 每个目录应启用关联的模块并使用模板的配置

### 需求：模板导出和导入

系统应支持将模板导出到文件和从文件导入模板。

#### 场景：通过文件共享模板

- **当** 用户将模板导出到文件
- **那么** 系统应创建包含模板定义的 JSON 文件
- **并且** 文件应可与其他用户共享

#### 场景：从文件导入模板

- **当** 用户从文件导入模板
- **那么** 系统应验证文件格式
- **并且** 模板应添加到用户 local 模板中

### 需求：模板预览

系统应提供应用模板时将创建的目录预览。

#### 场景：模板应用预览

- **当** 用户选择要应用的模板
- **那么** 系统应显示将要创建的目录树视图
- **并且** 预览应显示每个目录的模块和配置

### 需求：应用前修改模板

系统应允许在应用前修改模板配置。

#### 场景：应用前自定义模板

- **当** 用户应用模板
- **那么** 用户应能够在创建前修改目录名称
- **并且** 用户应能够调整模块配置
- **并且** 用户应能够排除特定目录

## 修改的需求

无。

## 移除的需求

无。
