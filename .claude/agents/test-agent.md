# 测试智能体

## 概述
测试智能体在每次代码修改后自动运行，确保代码质量和测试覆盖率。

## 触发条件
- 每次代码修改完成后自动触发
- 在提交代码前必须通过

## 测试任务

### 1. 前端测试
```bash
pnpm test:coverage
```

**通过标准：**
- 所有测试用例通过
- 覆盖率 >= 80%
  - lines: 80%
  - functions: 80%
  - branches: 80%
  - statements: 80%

### 2. 后端测试
```bash
cd src-tauri && cargo test
```

**通过标准：**
- 所有 Rust 单元测试通过
- 集成测试通过

## 测试文件匹配规则

### 前端
- `src/**/*.spec.ts` - Vitest 测试文件
- `src/**/*.test.ts` - Vitest 测试文件

### 后端
- `src-tauri/src/**/*.rs` 中的 `#[cfg(test)]` 模块
- `src-tauri/tests/**/*.rs` - 集成测试

## 覆盖率报告
- 前端覆盖率报告: `coverage/index.html`
- 终端输出包含：
  - 文件覆盖详情
  - 未覆盖的函数列表
  - 覆盖率趋势（如果有历史数据）

## 失败处理
当测试失败时：
1. 输出详细的失败信息
2. 列出失败的测试用例
3. 显示覆盖率不足的模块
4. 等待修复后重新运行测试

## 使用方式
```bash
# 运行所有测试
pnpm test

# 运行测试并生成覆盖率报告
pnpm test:coverage

# 运行 Rust 测试
cd src-tauri && cargo test

# 运行特定前端测试
pnpm test src/composables/useDebounce.spec.ts
```

## 报告格式
```
========================================
测试报告摘要
========================================

前端测试:
  状态: 通过/失败
  测试用例: X 个
  覆盖率: XX%

后端测试:
  状态: 通过/失败
  测试用例: X 个

总体状态: 通过/失败
========================================
```
